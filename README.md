# Порівняння ефективності жадібного алгоритму та алгоритму динамічного програмування

## Часова складність

- Жадібний алгоритм (find_coins_greedy): O(n), де n — кількість різних монет.
- Алгоритм динамічного програмування (find_min_coins): O(n * m), де n — кількість різних монет, а m — цільова сума.

## Ефективність при великих сумах:

- Час виконання динамічного програмування значно зростає.
- Час виконання жадібного алгоритму залишається швидким.

# Висновок:

Обидва алгоритма видають однаковий результат, але жадібний алгоритм має значно меньший час виконання, обобливо для
великих сум. Алгоритм динамічного програмування може знайти оптимальне рішення, але є повільнішим, бо час виконання
збільшується пропорційно сумі, яку потрібно розбити на монети, та кількості номіналів монет.
